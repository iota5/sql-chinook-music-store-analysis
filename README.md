# Chinook Music Store Analysis (MySQL)

## Objective
To analyze sales and customer data from the Chinook Music Store database using MySQL, with the goal of understanding purchasing behavior, customer lifecycle patterns, revenue drivers, and areas for business growth and retention.

## Tools Used
- MySQL
- Advanced SQL (CTEs, subqueries, joins, aggregations)
- Presentation & reporting tools for insights and recommendations

## Dataset Overview
The Chinook database is a relational music store dataset containing information about customers, invoices, invoice items, tracks, albums, artists, genres, and employees.

## Key Business Questions Addressed
- Which products, genres, and artists contribute most to revenue?
- How does customer spending behavior change over time?
- Which customers show signs of churn or reduced engagement?
- What regions and markets perform better or worse?
- Which customer segments should be prioritized for retention?

## SQL Concepts Applied
- Multi-table joins
- Subqueries (correlated and non-correlated)
- Common Table Expressions (CTEs)
- Case Statemts
- Window/Analytics Fucntions
- Aggregations and grouping
- Customer lifecycle and segmentation logic

## Analysis & Insights
- Identified high-value customers and at-risk segments based on spending patterns
- Analyzed genre- and product-level revenue contribution
- Examined regional performance differences to guide market strategy
- Derived actionable insights to support advertising prioritization and retention efforts

## Deliverables
- SQL file containing all business questions and corresponding MySQL queries
- Business presentation summarizing insights and recommendations
- Detailed analytical report with explanations, outputs, and visuals
- Screenshots of query outputs for reference and validation

## Notes on Outputs
Query results are generated dynamically by executing the provided SQL queries on the Chinook database in MySQL. Screenshots are included to demonstrate representative outputs.
